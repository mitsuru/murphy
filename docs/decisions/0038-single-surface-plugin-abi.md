# ADR 0038: Single-surface plugin ABI — `#[repr(C)]` arena direct-read

**Status**: Accepted (2026-05-22)
**Issue**: murphy-9cr.13 (parent epic: murphy-9cr)
**Related**: ADR 0002 (mruby bridge), ADR 0004 (trust model), ADR 0019 (native primitive IDL), ADR 0033 (plugin ABI v1 option metadata), ADR 0034 (no synthesized node dispatch), ADR 0035 (const-based `Cop` trait)
**Supersedes**: ADR 0031 (file-level native plugin pack ABI)
**Depends on**: ADR 0037 (arena, parser-shaped, typed AST)

## Context

ADR 0031 defined a Phase 8 file-level native plugin pack ABI. The
plugin-reboot design (`docs/plans/2026-05-22-plugin-reboot-design.md` §0)
treats the current plugin implementation — `MurphyNodeContext`,
`MurphyPluginCopV1`, the present `Cop` trait — as **pre-ABI-freeze spike
code**: no external consumer depends on it, and no back-compat or migration
shim is owed.

ADR 0037 makes the core AST an owned, flat, POD arena that Murphy controls
end to end. That removes the blocker that forced ADR 0031 to be file-level
(a borrowed `ruby_prism::Node` could not cross the ABI) and makes a greenfield
redesign of the plugin ABI both possible and cheap. This ADR formalizes the
**plugin ABI** half of the reboot. The core AST half is ADR 0037.

## Decision

1. **Single surface.** `murphy-plugin-api` is the *one* surface through
   which any cop reads the AST. murphy-core's standard cops and external
   `.so` plugins compile against the same crate. There is no second,
   internal-only AST API.

2. **Direct-read context.** A cop callback receives a `Cx<'a>` that reads
   the arena directly: a raw `*const AstNode`, a `*const NodeId` list
   table, and an interner reference. Tree traversal and `NodeKind` matching
   are pure memory reads — **zero FFI calls** on the hot path.

3. **Minimal `FnTable`.** Function pointers cover only operations that need
   murphy-core logic: `intern_to_str(Symbol) -> &str`,
   `emit_offense(range, message, …)`, `emit_edit(range, replacement)`,
   `comments() -> &[Comment]`, `raw_source(range) -> &str`. This is the
   concrete realization of ADR 0019's native primitive IDL.

4. **`.so` boundary.** murphy-core passes a plugin: the arena pointer +
   length, the `node_lists` table, the interner blob, the `FnTable`, and an
   ABI version. A plugin exports exactly one entry point,
   `murphy_plugin_register`, generated by `register_cops!`.

5. **Trait reshape.** `Cop` carries metadata only — `const`
   NAME / DESCRIPTION / severity / enabled / options. The const base
   established by ADR 0035 survives. `NodeCop` is the dispatch trait:
   `fn check(&self, node: NodeId, cx: &Cx)` plus a target-`NodeKind`
   declaration (`#[on_node]`). `FileCop` is **deleted**. `CallCop` is
   **merged into `NodeCop`** — a call cop is just a `NodeCop` on `Send`.

6. **`run_file` is removed.** The `FileCop` trait and the `MurphyRunFile`
   function pointer are deleted. There is no raw-whole-file escape hatch;
   `raw_source(range)` and `comments()` on the `FnTable` cover the
   legitimate source-text needs.

7. **Standard cops are a built-in plugin pack.** `murphy-rails` compiles
   against `murphy-plugin-api` exactly as an external plugin does, but is
   **statically linked** instead of `.so`-loaded. The only difference is
   discovery: a built-in list versus a `.so` scan.

8. **Greenfield v1.** The pre-reboot plugin ABI was never frozen. This
   ADR's ABI is the **fresh v1 starting point** — no version-bump dance, no
   compatibility shim. ABI-freeze discipline (additive-only changes,
   reserved padding) applies only from the v1 ship date forward, not to the
   reboot itself.

## Safety contract

- **No retained pointers.** `Cx<'a>` carries a lifetime that forbids a
  plugin from holding an arena pointer past the dispatch call. The arena is
  immutable during dispatch, owned by murphy-core, and outlives every
  plugin invocation.
- **No unwinding across the ABI.** A Rust panic must not cross the plugin
  ABI boundary (restated from ADR 0031).
- **Stateless cops.** Callbacks take no owned mutable state beyond the
  context; the typed AST view plus the emitter are everything a callback
  receives. This continues the stateless-cop philosophy of ADR 0035.
- **Trust model unchanged.** Native plugin packs are trusted code running
  with the full privileges of the Murphy process (ADR 0004, ADR 0031). The
  single-surface ABI is not a security boundary.

## Reasons

1. **One surface, one semantics.** A single API means one semantics to
   test and one contract to document, and standard cops dogfood the exact
   surface external plugins use.
2. **Direct-read is only possible because of ADR 0037.** A flat POD arena
   makes traversal a memory read; an FFI call per node access would
   dominate cost. The arena shape and the direct-read ABI are two halves of
   one decision.
3. **A minimal `FnTable` keeps the ABI stable.** Most of what a cop does is
   read memory; only emit / intern / source-text need host logic. A small
   function table is a small thing to keep frozen.
4. **Removing `run_file` collapses the trait surface.** File-level cops
   were the spike's escape hatch. With a real arena every cop is
   node-dispatched; deleting `FileCop` and folding `CallCop` into `NodeCop`
   continues the dispatch-surface tightening of ADR 0034.
5. **Greenfield is cheap now, expensive later.** The spike ABI has no
   in-tree external consumers (`murphy-rails` and the example pack go
   through murphy-core helpers, not this trait). This is the one chance to
   redesign without a migration burden.

## Alternatives considered

- **Keep ADR 0031's file-level C ABI and add a node-level ABI alongside
  it.** Rejected: two surfaces, two test suites, and the file-level escape
  hatch survives.
- **A Rust trait-object boundary instead of a C ABI direct-read.**
  Rejected: it relies on Rust ABI stability, which ADR 0031 already
  rejected for `.so` loading.
- **Keep `run_file` for raw-source cops.** Rejected: `raw_source(range)`
  plus `comments()` cover the legitimate needs, while an unstructured
  whole-file hook invites cops that bypass the AST entirely.

## Out of scope

- **Pattern DSL** (design doc §4 — `def_node_matcher!`, the B/C backends).
  A separate forthcoming ADR.
- **murphy-9cr epic restructure** (design doc §6 DAG). Operational
  sequencing, handled when the epic is re-planned.
- **The core AST representation** (the arena, `NodeKind`, traversal). ADR
  0037.
- **Migrating murphy-rails' 131 text-matching cops.** Design doc §6 handles
  the breakage with a temporary-disable mechanism plus a dedicated
  follow-up epic; not part of this ADR.

## Consequences

- **ADR 0031 is superseded.** The file-level native plugin pack ABI is
  replaced by this single-surface arena ABI. ADR 0031 receives a
  "Superseded by" status line.
- **ADR 0033's option-metadata concept survives, the struct is rebuilt.**
  The `MurphyCopOptionV1` schema concept carries forward, but the struct is
  re-implemented against the new ABI (design doc §6; murphy-9cr.2
  re-implement).
- **ADR 0035's const-based `Cop` survives**; `FileCop` is deleted and
  `CallCop` folds into `NodeCop`. `register_cops!` and
  `#[derive(CopOptions)]` re-target the reshaped traits.
- **ADR 0034 (no synthesized dispatch) is kept** and applies unchanged to
  arena dispatch.
- **ADR 0019's native primitive IDL is realized as the `FnTable`.**
- **murphy-rails breaks** when dispatch moves to the arena; the
  temporary-disable mechanism and the full-port follow-up epic (design doc
  §6) absorb that — out of scope here.
- **Depends on ADR 0037.** The arena surface must exist for "direct-read"
  to have a definition.
